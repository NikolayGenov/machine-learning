Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

For the performance_metric function in the code cell below, you will need to implement the following:

  • Use r2_score from sklearn.metrics to perform a performance calculation between y_true and y_predict.

from sklearn.metrics import r2_score 

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    score = r2_score(y_true,y_predict)
    
    # Return the score
    return score

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer: The benefit of such split is that we can use one part of avaiable to us data to test the prediction part of a model and measure its performance, while using the bigger part to build the model itself. If we don't do that - we might use all of our data to build the model, but then if we test it with the same model it will not show realistic results. Another option is to gather more new data, but that could be a hard task(time consuming and costly). The tradeoff we make is to use some part of our data as test data. If we choose too little data for that - we might not have a good test sample (e.g 3 samples for test out of 300 total)

Another thing is data leakage - if we do not separate the test set into validation and pure test - our test set (all test samples) could leak some knowledge into our model after many runs. So we need a validation and a separate hidden test set.

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer:

The grid search technique is that is use to search from a user defined grid of parameters (parameter space) of a given algorithm. The algorithm is tested with each of those parameters and the results are then ordered and the optimal can be easialy selected. It can be applied to optimize a learning algorithm by providing some sensible set of value for each parameter of the algorithm (e.g. C, kernel, gamma for Support Vector Classifier) and then the algorithm is left to calculate which is the best parameter of the given. Later only the best parameter is used for prediction when the model is actually used.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer:

K-fold cross validation is a technique in which the data is divided in K folds. Then for training we use K-1 folds and for testing we use 1 fold. We then iterate on the choice of that 1 fold for testing so we take each time a different group of those K fold. At each iteration we calculate the error for that fold. After we have iterated all the folds, we take the average of the errors of the folds and we get the cross validation error for that algorithm. Grid search searches for the parameters which produce the minimal cross validation error . Another note is that we should have a cross validation dataset and a separate test set, so there is not data leakage. It's good to give our grid search that cross validation set.

If we do a grid search on the same training and testing data which is split at the exactly same manner each time - the parameters that the grid search could learn could be specific to that specific split and could not generalize to the dataset at a whole. If we do K fold we avoid doing the same split twice so we can have a better certainty in the generalization of the data taken for the model and for the test set.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter=10, test_size=0.20, random_state=0)

    regressor = DecisionTreeRegressor()

    params = {'max_depth': range(1,11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor,params,scoring=scoring_fnc,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
